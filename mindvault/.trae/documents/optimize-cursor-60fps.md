# 自定义光标 60fps 动画优化计划

## 当前问题

### 问题 1：源素材只有 12 帧
.ani 文件本身只包含 12 帧动画数据，这是 Aemeath 主题光标的原始设计。无法从 12 帧凭空创造出 60 帧新内容。

### 问题 2：帧切换方式导致闪烁
当前使用 `<img src={frameSrc}>` 切换帧，每次改变 `src` 属性时浏览器需要重新解码图片，即使有缓存也会产生微小的闪烁/延迟，导致动画不流畅。

### 问题 3：速度计算错误
.ani 文件的 `nSpeed` 为 0（表示使用默认速率），当前代码 fallback 到 60 jiffies = 1000ms/帧，导致每帧停留 1 秒，动画极慢。

## 解决方案

### 核心思路：Canvas 渲染 + 帧插值

1. **Canvas 渲染替代 img 标签**：将所有帧预渲染到 Canvas，通过 `drawImage` 切换帧，消除 img src 切换的闪烁
2. **帧混合插值**：在 12 个关键帧之间进行 alpha 混合插值，生成中间过渡帧，使动画看起来更平滑
3. **60fps 渲染循环**：使用 `requestAnimationFrame` 驱动 60fps 渲染循环，而非 `setTimeout`
4. **修正动画速度**：当 nSpeed=0 时使用合理的默认帧率（约 8-10 jiffies/帧 ≈ 133-167ms/帧）

### 帧插值算法

对于 12 帧动画，在每两个关键帧之间插入 4 个中间帧：
- 帧 0.0 → 关键帧 0
- 帧 0.2 → 80% 帧0 + 20% 帧1
- 帧 0.4 → 60% 帧0 + 40% 帧1
- 帧 0.6 → 40% 帧0 + 60% 帧1
- 帧 0.8 → 20% 帧0 + 80% 帧1
- 帧 1.0 → 关键帧 1

这样 12 个关键帧 × 5 个子帧 = 60 个显示帧/周期，配合适当的播放速度即可实现流畅动画。

### 实现方式

使用 Canvas 的 `globalAlpha` 进行帧混合：
1. 清空 Canvas
2. 以 `globalAlpha = 1 - t` 绘制前一帧
3. 以 `globalAlpha = t` 绘制当前帧
4. `t` 从 0 到 1 线性插值

## 实施步骤

### 步骤 1：重写 CustomCursor 组件

- 使用 `<canvas>` 替代 `<img>`
- 预加载所有帧到 `HTMLImageElement[]`
- 使用 `requestAnimationFrame` 驱动 60fps 渲染循环
- 在关键帧之间进行 alpha 混合插值
- 修正默认动画速度

### 步骤 2：验证效果

- 确认动画流畅无闪烁
- 确认光标位置跟踪准确
- 确认光标类型切换正确
