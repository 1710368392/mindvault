## 二、用户故事

### 2.1 主用户故事（Epic）

```
作为 创意记录者
我想要 一个支持 Markdown 语法的富文本编辑器
以便于 我可以用格式化的方式记录和整理创意内容，
包括标题层级、列表、代码块、引用、表格等，
让我的创意不再是一堆纯文本，而是结构化的知识资产。
```

### 2.2 拆分子故事（User Stories）

#### US-01：快速录入支持 Markdown 编辑

```
作为 需要快速记录灵感的用户
我想要 在快速录入弹窗中使用 Markdown 编辑器而不是纯文本框
以便于 我在记录灵感时就能直接使用格式化语法（如 # 标题、- 列表、```代码块）
验收标准：
- QuickCapture 弹窗中的内容区域替换为 MDEditor 组件
- 默认显示"编辑+预览"分屏模式
- 支持工具栏快捷操作（加粗、斜体、标题、列表、链接、图片、代码块）
- 支持 Ctrl+S 快速保存
- 支持粘贴图片自动转为 base64 或上传
- 编辑器高度自适应（最小 150px，最大 300px）
- 保存时 contentFormat 自动标记为 'markdown'
```

#### US-02：浮动编辑器增强 Markdown 体验

```
作为 需要深度编辑创意的用户
我想要 浮动编辑器中的 Markdown 编辑器具备完整的写作辅助能力
以便于 我可以专注于创作而不会被编辑器的限制打断
验收标准：
- 全屏模式下编辑器高度撑满可用区域
- 工具栏增加"专注模式"按钮（隐藏预览区，仅显示编辑区）
- 工具栏增加"大纲导航"按钮（提取标题生成目录）
- 实时字数统计（中文按字符、英文按单词）
- 支持键盘快捷键：Ctrl+B 加粗、Ctrl+I 斜体、Ctrl+K 插入链接
- 图片拖拽/粘贴后正确嵌入 Markdown（![alt](url)）
- 自动保存草稿（每 30 秒或失焦时保存到 localStorage）
- 深色主题下编辑器配色与全局主题一致
```

#### US-03：Markdown 内容的统一渲染

```
作为 浏览创意内容的用户
我想要 所有展示创意内容的地方都能正确渲染 Markdown 格式
以便于 无论我在哪里查看创意，都能看到格式化后的效果
验收标准：
- CardPreview 弹窗中 Markdown 正确渲染（✅ 已完成）
- 首页"最近创意"列表中摘要截断不破坏 Markdown 结构
- 搜索结果列表中内容高亮不破坏 Markdown 渲染
- 看板便利贴卡片上支持简短 Markdown 渲染（标题、加粗、列表）
- 导出为 HTML/PDF 时保留 Markdown 格式
- contentFormat 字段为 'markdown' 时走渲染管线，'plain' 时走纯文本管线
```

#### US-04：模板与 Markdown 协同

```
作为 使用模板创建创意的用户
我想要 从模板库选择模板时，模板内容能以 Markdown 格式预填充到编辑器
以便于 我可以直接在模板基础上修改，而不需要重新排版
验收标准：
- 模板的 config 字段中的 content 是 Markdown 格式的文本
- 选择模板后打开 QuickCapture，内容区域预填充模板的 Markdown 文本
- MDEditor 正确解析并渲染模板中的 Markdown 格式
- 用户修改后的内容仍然保持 Markdown 格式保存
```

## 四、API 接口清单

### 4.1 前端 → 后端（IPC）

本功能**不需要新增 IPC 接口**。现有的 API 已经满足需求：

| 接口 | 方法 | 用途 | 现状 |
|------|------|------|------|
| `creativity.create` | POST | 创建创意（含 markdown 内容） | ✅ 已有 |
| `creativity.update` | PUT /:id | 更新创意（含 markdown 内容） | ✅ 已有 |
| `creativity.get` | GET /:id | 获取创意详情 | ✅ 已有 |
| `media.save` | POST | 上传媒体文件（图片/音频） | ✅ 已有 |
| `template.list` | GET | 获取模板列表 | ✅ 已有 |

**需要确认的数据字段**：

```typescript
// 创建/更新创意时的数据结构（已有，需确认兼容性）
interface CreativityCreateInput {
  title: string;
  content: string;           // Markdown 源码
  type: 'text' | 'image' | 'audio' | 'link' | 'video';
  subtype?: WritingSubtype;
  contentFormat: 'plain' | 'markdown';  // 关键！标记为 'markdown'
  wordCount?: number;
  priority?: number;
  emojiReaction?: string | null;
  tags?: string[];
  cardStyle?: string | null;
}
```

### 4.2 新增前端内部接口（Utils/Hooks）

| 编号 | 名称 | 类型 | 说明 |
|------|------|------|------|
| FE-01 | `useAutoSave` | Hook | 自动保存逻辑：防抖 30s 或失焦时触发 localStorage 草稿保存 |
| FE-02 | `useMarkdownOutline` | Hook | 解析 Markdown 文本，提取标题层级生成目录树 |
| FE-03 | `useMarkdownWordCount` | Hook | 计算 Markdown 纯文本的字数（排除标记符号） |
| FE-04 | `handleImagePaste` | Function | 处理图片粘贴事件：转 base64 或调用 media.save 上传 |
| FE-05 | `MDEditorTheme` | CSS Module | 自定义 MDEditor 主题变量，适配脑洞集 5 套主题色系 |

### 5.2 MDEditor 配置方案

```typescript
// 推荐的 MDEditor 统一配置
const mdEditorConfig = {
  // 视图模式
  preview: 'live',          // 实时预览（编辑即渲染）
  
  // 高度
  height: 200,              // QuickCapture 最小高度
  minHeight: 150,

  // 工具栏

    ['unorderedList', 'orderedList', 'check'],

  // 主题适配
  dataColorMode: getCurrentTheme(), // 'light' | 'dark'

  // 其他
  autoFocus: false,
  tabSize: 2,
  previewOptions: {
    remarkPlugins: [remarkGfm],  // 启用 GFM
  },

```

### 5.3 主题适配策略

脑洞集有 5 套主题，MDEditor 只支持 light/dark 两种模式：

| 脑洞集主题 | MDEditor 映射 | 处理方式 |
|-----------|---------------|---------|
| light（经典白） | `light` | 直接映射 |
| dark（经典黑） | `dark` | 直接映射 |
| morandi-warm（暖调莫兰迪） | `light` + 自定义 CSS 变量覆盖 |
| morandi-cool（冷调莫兰迪） | `light` + 自定义 CSS 变量覆盖 |
| morandi-nature（自然莫兰迪） | `light` + 自定义 CSS 变量覆盖 |

**CSS 变量覆盖示例**：
```css
/* morandi-warm 主题下的 MDEditor 适配 */
[data-theme="morandi-warm"] .w-md-editor {
  --color-fg-default: #5d4037;
  --color-bg-page: #fff8e1;
  --color-border-default: #d7ccc8;
}
```

### 5.4 图片处理流程

```
用户粘贴/拖拽图片
       │
       ▼
┌──────────────┐
│  onPaste /   │  捕获 ClipboardEvent / DragEvent
│  onDrop      │
└──────┬───────┘
       │
       ├─ 方案 A：base64 内嵌（< 512KB）
       │  → 转为 data:image/png;base64,...
       │  → 嵌入 Markdown: ![alt](data:image/...)
       │  → 适合：快速记录、离线使用
       │
       └─ 方案 B：上传至 media 表（>= 512KB）
          → 调用 api.media.save({ fileName, fileType, data })
          → 返回 filePath
          → 嵌入 Markdown: ![alt](file://local-path)
          → 适合：大图片、跨创意复用
```

**推荐**：第一阶段全部使用方案 A（base64），后续优化为大文件走方案 B。
