# 实机测试能力说明与方案

## 我能做什么

是的，我可以对你的 Electron 本地应用进行实机测试。具体能力如下：

### ✅ 可以做的测试

| 测试类型 | 实现方式 | 示例 |
|---------|---------|------|
| **UI 交互测试** | Playwright 连接 Electron 的 CDP 端口 (9222) | 点击按钮、填写表单、切换页面 |
| **截图验证** | Playwright 截图保存到本地 | 验证 UI 布局、颜色、样式 |
| **页面内容读取** | Playwright 读取 DOM 文本和结构 | 验证文字内容、数据展示 |
| **状态持久性测试** | 操作 → 导航离开 → 返回 → 验证状态 | 灵感闪回音量保持、画布编辑保持 |
| **控制台日志监控** | 读取 Electron 主进程和渲染进程日志 | 检查 IPC 调用、错误信息 |
| **数据库验证** | 通过 IPC 调用或直接读取 SQLite | 验证数据是否正确持久化 |
| **多步骤流程测试** | Playwright 持久化会话模式 | 模拟完整用户操作流程 |

### ❌ 无法做的测试

| 限制 | 原因 |
|------|------|
| 原生对话框交互 | Electron 原生菜单/对话框不在 Web 渲染层 |
| 视觉审美判断 | 我无法像人一样判断"好不好看" |
| 音频/视频内容验证 | 无法听/看媒体内容本身 |
| 系统级操作 | 文件选择器、系统托盘等 |

## 测试方法

### 连接方式

Electron 启动时已开启 `--remote-debugging-port=9222`，Playwright 可通过 CDP 协议连接：

```javascript
const browser = await chromium.connectOverCDP('http://localhost:9222');
const context = browser.contexts()[0];
const page = context.pages()[0];
```

### 典型测试流程

1. 连接到已运行的 Electron 实例
2. 获取页面结构 (`helpers.describePageForAI`)
3. 执行操作（点击、输入、导航）
4. 截图记录
5. 验证结果（读取 DOM、检查日志）

## 针对当前待修复问题的测试计划

### 灵感闪回视频音量持久性测试

**目标**：验证在灵感闪回中调整视频音量后，切换页面再切回，音量设置是否保持

**步骤**：
1. 连接 Electron，导航到首页
2. 等待灵感闪回加载，如遇视频类创意则调整音量
3. 记录当前音量值
4. 导航到其他页面（如设置页）
5. 导航回首页
6. 验证灵感闪回视频音量是否与之前一致
