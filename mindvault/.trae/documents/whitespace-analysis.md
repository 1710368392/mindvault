# 两侧留白产生原因分析

## 当前结构树

```
┌─── Carousel（Ant Design 轮播，第911行）───────────────────────────────┐
│  style: flex: 1（撑满卡片 body）                                       │
│  内部：Ant Design 自动生成 slick 容器（宽度 100%）                        │
│                                                                       │
│  ┌─── slick 内部 slide 容器（Ant Design 生成，宽度 100%）──────────┐  │
│  │                                                                  │  │
│  │  ┌─── div key={fb.id}（第945行）─────────────────────────────┐ │ │
│  │  │  width: fit-content, maxWidth: 100%, margin: 0 auto        │ │ │
│  │  │ （这个容器收缩了，但上面的 slick 容器还是 100%，留白在这里）│ │ │
│  │  │                                                            │ │ │
│  │  │  ┌─── .flashback-card（第956行）───────────────────────┐ │ │ │
│  │  │  │  width: fit-content, maxWidth: 100%, margin: 0 auto │ │ │ │
│  │  │  │  （跟着视频走）                                      │ │ │ │
│  │  │  │                                                    │ │ │ │

│  │  │  └───────────────────────────────────────────────────────┘ │ │ │
│  │  │                                                            │ │ │
│  │  └─────────────────────────────────────────────────────────────┘ │ │
│  │                                                                  │ │
│  └───────────────────────────────────────────────────────────────────┘ │
│                                                                       │
└───────────────────────────────────────────────────────────────────────┘
```

## 留白产生的根源

**问题**：Ant Design Carousel（基于 react-slick）内部有多层固定宽度的容器：
1. Carousel 自身宽度 100%（撑满卡片）
2. 内部 `.slick-track` 或 `.slick-slide` 等容器也是 100%
3. 我们在 `div key={fb.id}` 这层设置 `width: fit-content` 收缩了，但它的**父容器（Ant Design 自动生成）还是 100%**，所以这个 253px 的 div 在一个 400px 的容器里居中，左右就各有 70px+ 留白！

## 解决方案

需要给 Carousel 组件也加上 `width: fit-content` 和 `margin: 0 auto`，并且可能需要覆盖 Ant Design Carousel 的内部样式，让它的 track/slide 容器也收缩。
