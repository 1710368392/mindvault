# 当前视频区域的层级结构（说人话版）

## 视频和标题确实是同一层级的！

它们都是 `.flashback-card` 这个卡片容器的"孩子"，是平级关系。

打个比方：`.flashback-card` 就是一个盒子，标题和视频都放在这个盒子里，并排放着。

## 现在的布局逻辑

```
┌─────────────── .flashback-card（卡片盒子）──────────────────┐
│  width: fit-content（盒子宽度跟着内容走）                      │
│  margin: 0 auto（盒子在轮播区域里居中）                        │
│                                                              │
│  ┌─ 类型标签行 ─────────────────────────────────────────┐    │
│  │ 🎬 视频 - 3分钟前                                      │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                              │
│  ┌─ 标题 ──────────────────────────────────────────────┐    │
│  │ 这是一个竖屏视频的标题                                  │    │
│  └──────────────────────────────────────────────────────┘    │
│                                                              │
│  ┌─ 视频 ─────────────────┐                                 │
│  │                         │                                 │
│  │    （视频画面 253×450）  │                                 │
│  │                         │                                 │
│  └─────────────────────────┘                                 │
│                                                              │
└──────────────────────────────────────────────────────────────┘
```

## 关键点

1. **卡片宽度跟着视频走**：卡片用了 `width: fit-content`，所以卡片有多宽取决于里面最宽的东西是什么
2. **标题会自动换行**：标题是文字，如果卡片比标题窄，文字会自动折行
3. **视频决定了卡片宽度**：竖屏视频缩放后 253px 宽 → 卡片也是 253px 宽（加 padding）
4. **整个卡片居中**：卡片在轮播区域里通过 `margin: 0 auto` 居中

## 竖屏视频 vs 横屏视频

**竖屏视频（9:16）**：
- 视频缩放后窄（253px）
- 卡片也窄（269px）
- 卡片在轮播区域里居中，两侧有空间

**横屏视频（16:9）**：
- 视频缩放后宽（798px），但被 maxWidth 限制为 384px
- 卡片撑满轮播区域（400px）
- 两侧没有多余空间
