# 灵感闪回 - 后台音视频自动播放优化方案

## 问题分析

### 现状
灵感闪回使用 Ant Design 的 `Carousel` 组件加载 3 条随机创意内容，配置了 `autoplay`（8秒切换）和 `effect="fade"` 淡入淡出效果。

**核心问题 1：** Carousel 的所有 slide 都在 DOM 中同时渲染（只是通过 CSS 隐藏非活跃页），而视频和音频元素设置了 `autoPlay={saved?.playing !== false}`（默认为 `true`），导致后台 slide 中的音视频也会自动播放，产生噪音。

**核心问题 2：** 固定 8 秒切换，如果音视频播放时长超过 8 秒，会被强制切走，体验不佳。应等音视频播放完毕再切换。

**附加问题：** 当前只用了单个 `flashbackVideoRef` 和单个 `flashbackAudioRef`，3 个 slide 共享同一个 ref，导致只有最后一个渲染的媒体元素能被 ref 引用到，其他媒体元素完全失控。

## 解决方案

### 核心思路
1. **关闭 Carousel 内置 autoplay，改用自定义定时器管理** — 根据当前 slide 的媒体类型动态决定切换时机
2. **追踪当前活跃的 Carousel 页索引** — 使用 Carousel 的 `afterChange` 回调
3. **用 ref Map 替代单个 ref** — 为每个媒体元素分配独立的 ref，实现精确控制
4. **仅活跃页的媒体允许播放** — 切页时暂停旧页媒体、恢复新页媒体
5. **智能切换策略** — 图片/文本 slide 用 8 秒定时器；音视频 slide 等播放完毕再切换
6. **区分自然结束与用户暂停** — 通过 `onEnded` 事件触发切换，用户手动暂停不触发

### 具体实现步骤

#### 步骤 1：新增状态和 ref

```typescript
// 活跃页索引
const [activeFlashbackIndex, setActiveFlashbackIndex] = useState(0);

// 替换 flashbackVideoRef 和 flashbackAudioRef，改为 ref Map
const flashbackMediaRefs = useRef<Record<string, HTMLVideoElement | HTMLAudioElement | null>>({});

// Carousel ref，用于程序化切换 slide
const flashbackCarouselRef = useRef<any>(null);

// 自动切换定时器
const flashbackAutoTimerRef = useRef<ReturnType<typeof setTimeout> | null>(null);

// 用户手动暂停标记（区分自然结束 vs 用户暂停）
const flashbackUserPausedRef = useRef<Record<string, boolean>>({});
```

#### 步骤 2：实现智能切换逻辑

核心函数：根据当前 slide 的媒体类型，决定何时切换到下一页：

```typescript
const scheduleFlashbackSwitch = useCallback(() => {
  // 清除旧定时器
  if (flashbackAutoTimerRef.current) {
    clearTimeout(flashbackAutoTimerRef.current);
    flashbackAutoTimerRef.current = null;
  }

  const currentFb = flashbacks[activeFlashbackIndex];
  if (!currentFb) return;

  if (currentFb.type === 'video' || currentFb.type === 'audio') {
    // 音视频 slide：不设定时器，等 onEnded 事件触发切换
    // 但如果媒体时长 <= 8秒，仍用 8 秒定时器兜底
    const mediaEl = flashbackMediaRefs.current[currentFb.id];
    if (mediaEl && mediaEl.duration && isFinite(mediaEl.duration) && mediaEl.duration <= 8) {
      flashbackAutoTimerRef.current = setTimeout(goToNextFlashback, 8000);
    }
    // 否则：等 onEnded 回调触发切换，不设定时器
  } else {
    // 图片/文本 slide：8 秒定时器切换
    flashbackAutoTimerRef.current = setTimeout(goToNextFlashback, 8000);
  }
}, [flashbacks, activeFlashbackIndex]);

const goToNextFlashback = useCallback(() => {
  if (flashbackCarouselRef.current && flashbacks.length > 1) {
    flashbackCarouselRef.current.next();
  }
}, [flashbacks.length]);
```

#### 步骤 3：媒体 onEnded 回调 — 音视频播放完毕触发切换

```typescript
const handleFlashbackMediaEnded = useCallback((fbId: string) => {
  // 只有当前活跃页的媒体结束才触发切换
  const currentFb = flashbacks[activeFlashbackIndex];
  if (currentFb && currentFb.id === fbId) {
    // 延迟一小段时间再切换，给用户一个"播放结束"的感知
    flashbackAutoTimerRef.current = setTimeout(goToNextFlashback, 1500);
  }
}, [flashbacks, activeFlashbackIndex, goToNextFlashback]);
```

#### 步骤 4：用户暂停/播放追踪

```typescript
// 用户通过控件手动暂停时标记
const handleFlashbackMediaPause = useCallback((fbId: string) => {
  flashbackUserPausedRef.current[fbId] = true;
  // 用户暂停时，清除自动切换定时器（不切换）
  if (flashbackAutoTimerRef.current) {
    clearTimeout(flashbackAutoTimerRef.current);
    flashbackAutoTimerRef.current = null;
  }
}, []);

// 用户手动恢复播放时，重新安排切换
const handleFlashbackMediaPlay = useCallback((fbId: string) => {
  flashbackUserPausedRef.current[fbId] = false;
  // 重新安排切换（音视频等 onEnded，图片/文本用定时器）
  scheduleFlashbackSwitch();
}, [scheduleFlashbackSwitch]);
```

#### 步骤 5：活跃页切换时的媒体控制 useEffect

```typescript
useEffect(() => {
  flashbacks.forEach((fb, index) => {
    const mediaEl = flashbackMediaRefs.current[fb.id];
    if (!mediaEl) return;
    if (index === activeFlashbackIndex) {
      // 活跃页：恢复播放（除非用户手动暂停过）
      if (!flashbackUserPausedRef.current[fb.id]) {
        const saved = flashbackPlayingState.current[fb.id];
        if (saved) {
          mediaEl.currentTime = saved.currentTime;
        }
        mediaEl.play().catch(() => {});
      }
    } else {
      // 非活跃页：暂停并保存状态
      if (!mediaEl.paused) {
        flashbackPlayingState.current[fb.id] = {
          playing: true,
          currentTime: mediaEl.currentTime,
        };
        mediaEl.pause();
      }
    }
  });
  // 活跃页变化时，安排下一次切换
  scheduleFlashbackSwitch();
  // 清理：组件卸载或依赖变化时清除定时器
  return () => {
    if (flashbackAutoTimerRef.current) {
      clearTimeout(flashbackAutoTimerRef.current);
    }
  };
}, [activeFlashbackIndex, flashbacks, scheduleFlashbackSwitch]);
```

#### 步骤 6：修改 Carousel 配置

关闭内置 autoplay，改用自定义切换逻辑：

```tsx
<Carousel
  ref={flashbackCarouselRef}
  autoplay={false}
  effect="fade"
  dots={flashbacks.length > 1}
  adaptiveHeight
  style={{ flex: 1 }}
  afterChange={(index) => setActiveFlashbackIndex(index)}
>
```

#### 步骤 7：修改媒体元素渲染

- 将 `ref={flashbackVideoRef}` / `ref={flashbackAudioRef}` 改为 `ref={(el) => { flashbackMediaRefs.current[fb.id] = el; }}`
- 将 `autoPlay={saved?.playing !== false}` 改为 `autoPlay={false}`（由 useEffect 控制播放）
- 添加 `onEnded={() => handleFlashbackMediaEnded(fb.id)}` 回调
- 添加 `onPause={() => handleFlashbackMediaPause(fb.id)}` 回调
- 添加 `onPlay={() => handleFlashbackMediaPlay(fb.id)}` 回调
- 在 `onLoadedMetadata` 中，仅当该 slide 是活跃页时才触发播放

#### 步骤 8：修改 loadFlashback 函数

```typescript
const loadFlashback = async () => {
  // 清除自动切换定时器
  if (flashbackAutoTimerRef.current) {
    clearTimeout(flashbackAutoTimerRef.current);
    flashbackAutoTimerRef.current = null;

      mediaRef.pause();
    }
  });
  // 重置用户暂停标记
  flashbackUserPausedRef.current = {};
  // 加载数据...
  setFlashbacks(valid);
  setActiveFlashbackIndex(0);
  // ...
};
```

#### 步骤 9：删除不再需要的旧 ref

删除 `flashbackVideoRef` 和 `flashbackAudioRef` 的声明，以及所有引用它们的代码，替换为 `flashbackMediaRefs`。

## 切换策略总结

| 当前 slide 类型 | 切换时机 |
|---|---|
| 图片 / 文本 / 链接 | 8 秒定时器 |
| 音视频（时长 ≤ 8秒） | 8 秒定时器兜底 + onEnded 触发 |
| 音视频（时长 > 8秒） | 仅 onEnded 触发（播放完毕后延迟 1.5 秒切换） |
| 用户手动暂停 | 不切换，等待用户恢复播放 |
| 用户恢复播放 | 重新安排切换策略 |

## 预期效果
- 只有当前可见的 Carousel 页中的音视频会播放，消除后台噪音
- 音视频播放完毕后才切换，不会被 8 秒定时器强制切走
- 用户手动暂停不会触发切换，恢复播放后继续等待播放完毕
- 图片/文本 slide 仍保持 8 秒自动切换
- 切页时旧页媒体自动暂停，新页媒体自动恢复
